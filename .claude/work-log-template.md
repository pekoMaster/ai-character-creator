# 工作記錄規範

## 📌 重要提醒
**每次工作都必須創建完整的工作記錄！**

用戶明確要求：每次工作結束後都要寫入工作記錄，不得遺漏。

## 工作記錄模板

創建於：`WORK_LOG_YYYY-MM-DD_HHMM.md`

### 1. 工作概述
- 工作日期時間：
- 工作類型：[新功能/Bug修復/優化/重構/文檔]
- 主要目標：

### 2. 完成的任務
- [ ] 任務 1
- [ ] 任務 2
- [ ] 任務 3

### 3. 技術細節
#### 修改的文件
- 文件路徑 (行號範圍)
  - 修改內容描述
  - 修改原因

#### 新增的功能/代碼
- 功能名稱
  - 實現位置
  - 技術方案
  - 關鍵代碼片段

#### 修復的問題
- 問題描述
  - 根本原因
  - 修復方案
  - 影響範圍

### 4. Git 提交記錄
```
commit hash:
commit message:
files changed:
```

### 5. 測試狀態
- [ ] 代碼審查通過
